Introduccion

Que es SQL?

  • SQL significa Lenguaje Estructurado de Consultas por sus siglas en ingles
  • SQL permite acceder y manipular bases de datos
  • SQL es un estandar ANSI

Que puede hacer SQL?

  • Puede ejecutar consultas a una base de datos
  • Puede devolver informacion de una base de datos
  • Puede insertar registros en una base de datos
  • Puede actualizar registros en una base de datos
  • Puede eliminar registros en una base de datos
  • Puede crear nuevas bases de datos
  • Puede crear nuevas tablas en una base de datos
  • Puede crear procedimientos almacenados en una base de datos
  • Puede crear vistas en una base de datos
  • Puede colocar permisos en tablas, procedimientos y vistas

Que es RDBMS?

  • RDBMS significa Sistema de Gestion de Bases de Datos Relacionales por sus siglas en ingles
  • La informacion en un RDBMS es almacenada en objetos llamados tablas
  • Una tabla es una coleccion de datos relacionados y esta formada por filas y columnas

Llave Primaria

  • Asegura que los registros sean unicos evitando ambiguedad

  • Se utiliza para hacer referencia a otras tablas

paso 3: Crear relaciones entre tablas

Relación: Es una asociación entre dos tabla. Muestra la interacción que hay entre ellas.


Relación uno a muchos


Un registro en una tabla A se relaciona con cero o muchos registros en una tabla B. Pero los registros de B solamente se relacionan con un registro en A.


Ej. Un profesor puede impartir varias clases pero una clase puede ser impartida por un unico profesor

Relacion muchos a muchos


Una tabla A se puede relacionar con cero o con muchos registros en B y viceversa. Este tipo de relación no puede existir por lo que debe ser disuelta.


Para ello se crea una tabla intermedia con las llaves primarias de las tablas a disolver. Se pueden agregar otros atributos a la nueva tabla.


Ej. Una orden puede incluir varios productos y un producto puede estar incluido en varias ordenes, por ello se crea una tabla, la cual llamaremos detalles de la orden.

Relacion uno a uno


Un registro de una tabla A se relaciona con solo un registro en una tabla B y viceversa.


Ej. Un producto puede tener un detalle y ese detalle pertenece unicamente a un producto

¿Qué es DDL?

Son consultas que se realizan para crear una base de datos y manipular su estructura, es decir crear, modificar y eliminar sus tablas.

>>

Ejemplo

Constraints

Especifican reglas para los valores en una tabla.


Not Null


Una columna no puede tener valores nulos

Unique


Asegura que cada registro en esa columna sea único. Puede existir más de una.

Primary Key


Identifica un registro de manera única. Esto ayuda a encontrar el registro de manera más fácil y rápida. No puede repetirse ni ser nula.



Una llave primaria puede estar compuesta por varias columnas


Foreign Key


Hace referencia de una tabla a otra. Una llave foránea en una tabla apunta a la llave primaria de otra tabla.

DML (Data Manipulation Language)

¿Qué es DML?

Son consultas que se realizan para manipular los datos dentro de la base de datos, es decir insertar datos a la base de datos, eliminarlos, actualizarlos o seleccionarlos.

>>

Insertar Datos

Sintaxis
INSERT INTO nombreTabla (columna1, columna2, columna3,...)
VALUES(valor1, valor2, valor3,...);

Ejemplo
INSERT INTO Customers (
CustomerName, ContactName, Address, City, PostalCode, Country)
VALUES
('Cardinal','Tom B. Erichsen','Skagen 21','Stavanger','4006','Norway');

Para insertar datos tipo varchar, estos van encerrados en comillas simples. Los valores se irán ingresando en el mismo orden en el que se definieron las columnas; en el ejemplo ‘Cardinal’ se almacenará en CustomerName, y ‘4006’ en PostalCode.

<<
>>

Eliminar datos

Todos los datos

Sintaxis
DELETE FROM nombreTabla;

Ejemplo
DELETE FROM customers;

Datos específicos

Sintaxis
DELETE FROM nombreTabla WHERE
nombreColumna = algunValor;

Ejemplo
DELETE FROM customers
WHERE CustomerName= 'Alfreds Futterkiste' AND ContactName='Maria Anders' ;

En este ejemplo se eliminarán todos los clientes cuyo nombre del cliente sea ‘Alfreds Futterkiste’ y el nombre del contacto sea ‘María Anders’

<<
>>

Actualizar Datos

Sintaxis
UPDATE nombreTabla
SET columnaModificar1 = valor1, columnaModificar2 = valor2,...
WHERE algunaColumna = valor;

Ejemplo
UPDATE Customers
SET ContactName = 'Alfred Schmidt',
City = 'Frankfurt'
WHERE CustomerID = 1;

Si no se coloca la condición WHERE se actualizarán todos los registros en la tabla.
En el ejemplo se asigna el valor ‘Alfred Schmidt’ a los registros de la columna ContactName y el valor ‘Frankfurt’ a los registros en la columna city de la tabla Customers cuando el CustomerID sea igual a 1.

Joins

¿Qué es un join?

Sirven para combinar filas de dos o más tablas donde la condición se cumple, en base a campos en común entre ellos. Son las relaciones vistas en lecciones anteriores.

>>

Inner Join

Selecciona todas las filas de ambas tablas, mientras que haya una coincidencia entre las columnas en ambas tablas.

Sintaxis
SELECT nombreColumna(s)
FROM tabla1 INNER JOIN tabla2 ON tabla1.nombreColumnaCoincidir = tabla2.nombreColumnaCoincidir;

Ejemplo
SELECT Employee.Name, Employee.EmpId, DeptName FROM Employee INNER JOIN Dept ON Employee.DeptName=Dept.DeptName;
La tabla users y la tabla orders está relacionada por la columna users.id y orders.user_id, en el ejemplo devuelve el nombre de usuario y el item a comprar de los usuarios cuyo id coincida con el order.user_id.

<<
>>

Left Join

Retorna todas las filas de la tabla de la izquierda con las que coinciden de la tabla de la derecha. El resultado es null del lado derecho cuando no hay coincidencia.

Sintaxis
SELECT nombreColumna(s)
FROM tabla1 LEFT JOIN tabla2 ON tabla1.nombreColumnaCoincidir = tabla2.nombreColumnaCoincidir;

Ejemplo
SELECT users.Name, orders.item FROM Users LEFT JOIN Orders ON users.id=orders.user_id;
En el ejemplo devuelve todos los registros de la tabla users y los registros de la tabla orders que coincidan con los registros de la tabla users.

<<
>>

Right Join

Retorna todas las filas de la tabla de la derecha con las que coinciden de la tabla de la izquierda. El resultado es null del lado izquierdo cuando no hay coincidencia.

Sintaxis
SELECT nombreColumna(s)
FROM tabla1 RIGHT JOIN tabla2 ON tabla1.nombreColumnaCoincidir = tabla2.nombreColumnaCoincidir;

Ejemplo
SELECT users.Name, orders.item FROM Users RIGHT JOIN Orders ON users.id=orders.user_id;
En el ejemplo devuelve todos los registros de la tabla orders y los registros de la tabla users que coincidan con los registros de la tabla orders.

<<
>>

Full Outer Join

Retorna todos los valores de ambas tablas.

Sintaxis
SELECT nombreColumna(s)
FROM tabla1 FULL OUTER JOIN tabla2 ON tabla1.nombreColumnaCoincidir = tabla2.nombreColumnaCoincidir;

Ejemplo
SELECT users.Name, orders.item FROM Users FULL OUTER JOIN Orders ON users.id=orders.user_id;
En el ejemplo devuelve todos los registros de la tabla users y todos los registros de la tabla orders.

Subqueries

¿Qué es un subquery?

Es un query dentro de otro query. Los registros que devuelve la subconsulta son utilizados por la consulta principal como una condición.

Puede ser utilizada por las clausulas select, insert, update y delete. También puede utilizar los operadores <, >, =, >=, <=, etc.

>>

Reglas

  • Deben ir encerradas entre parentesis
  • Solo puede tener una columna en la clausula SELECT, a menos que hayan multiples columnas en el query principal para comparar los resultados.
  • No se puede utilizar ORDER BY en una subquery, sin embargo el query principal sí puede utilizarlo. En lugar de ORDER BY se puede utilizar GROUP BY.
  • Subqueries que devuelven multiples filas solo pueden ser utilizados por operadores que soportan multiples resultados como IN
  • La lista del select no puede incluir referencias a valores de tipo BLOB, ARRAY, CLOB y NCLOB.
  • Una subquery no puede ser utilizada directamente en una funcion set (update).
  • No se puede utilizar BETWEEN con la subquery, pero sí se puede utilizar dentro de ella.
<<
>>

Sintaxis
SELECT nombreColumna(s)
FROM tabla1 WHERE nombreColumna = (SELECT nombreColumna(s)
FROM tabla1 WHERE nombreColumna = valorComparar);

Ejemplo
SELECT * FROM Customers WHERE ID IN (SELECT ID FROM Customers WHERE salary > 4500);
En el ejemplo devuelve los registros de la tabla customers que tengan un salario > 4500. La subconsulta devuelve el ID del usuario verificando que el salario sea > 4500. Luego la consulta padre compara los IDs devueltos para retornar los registros.

<<
>>

Ejemplo
Mostrar el nombre del pais cuya poblacion sea mas grande que la poblacion de Rusia

SELECT name FROM world WHERE population > (SELECT population FROM world WHERE name = 'Russia');
La subconsulta devuelve la poblacion de Rusia. Luego la consulta padre compara la poblacion de los paises con la poblacion de Rusia.

<<
>>

Ejemplo
Mostrar el nombre de paises europeos con un GDP per capita mayor al de United Kingdom
GDP per capita = gdp/population

SELECT name FROM world WHERE gdp/population > (SELECT gdp/population FROM world WHERE name = 'United Kingdom') AND continent = 'Europe';

<<
>>

Ejemplo
¿Qué países tienen una población mayor a la de Canada y menor a la de Poland?

SELECT name,population FROM world WHERE population > (SELECT population FROM world WHERE name = 'Canada') AND continent < (SELECT population FROM world WHERE name = 'Poland');

Funciones

¿Qué es una funcion?

Son funciones predeterminadas que permiten realizar calculos en los datos.

>>

>>

Count

Retorna la cantidad de registros de determinada columna. Los valores nulos no son tomados en cuenta.

Sintaxis

Ejemplo

<<
>>

MAX

Retorna el valor mas grande de la columna seleccionada.

Sintaxis
SELECT MAX(nombreColumna) FROM nombreTabla;

Ejemplo
SELECT MAX(Price) FROM products;

<<
>>

MIN

Retorna el valor mas pequeño de la columna seleccionada.

Sintaxis
SELECT MIN(nombreColumna) FROM nombreTabla;

Ejemplo

<<
>>

SUM

Retorna la suma total de una columna numerica.

Sintaxis
SELECT SUM(nombreColumna) FROM nombreTabla;

Ejemplo

<<
>>

Group By

Generalmente las funciones AVG, SUM o COUNT deben incluir una sentencia Group By. Sirve para agrupar el resultado por una o mas columnas.

Sintaxis
SELECT SUM/AVG/COUNT(nombreColumna) FROM nombreTabla WHERE nombreColumna = valor GROUP BY nombreColumna;

Ejemplo
SELECT dept_id, SUM(salary)AS total_salaries FROM employees GROUP BY dept_id;

<<
>>

Having

Equivalente a la clausula WHERE. Se utiliza cuando la condición contiene las funciones AVG, SUM o COUNT.

Sintaxis
SELECT nombreColumna FROM nombreTabla GROUP BY nombreColumna HAVING SUM/AVG/COUNT(nombreColumna) = valor;

Ejemplo
SELECT COUNT(Id), country FROM customer GROUP BY country HAVING COUNT(Id)>10;

<<
>>

UCASE

Convierte el valor de la columna a mayúscula sostenida. SQL Server utiliza UPPER en lugar de UCASE.

Sintaxis

Ejemplo

<<
>>

LCASE

Convierte el valor de la columna a minúscula sostenida. SQL Server utiliza LOWER en lugar de LCASE.

Sintaxis

Ejemplo

<<
>>

MID

Extrae caracteres de una columna de texto. En SQL Server se utiliza SUBSTRING en su lugar y ORACLE utiliza SUBSTR. Se especifica el nombre de la columna a la que se aplicará la función, donde iniciará a sustraer caracteres y opcionalmente la cantidad de caracteres a devolver, si no se especifica retornará el resto del texto.

Sintaxis

Ejemplo
SELECT MID(City,1,4) AS ShortCity FROM customer;

<<
>>

LEN

Devuelve el largo de la cadena.

Sintaxis

Ejemplo

>>

Round

Redondea una columna numérica al número de decimales especificado.

Sintaxis

Ejemplo

>>

Now

Devuelve la fecha y hora actual del sistema.

Roles y Permisos

¿Qué es un rol?

Se utilizan para administrar más fácilmente los permisos en las bases de datos. Un rol es una entidad que agrupa varios usuarios con determinados permisos.
Existen dos tipos de roles en el nivel de base de datos: los roles fijos de base de datos que están predefinidos en la base de datos y los roles de base de datos definidos por el usuario que el usuario puede crear.

>>

Roles Fijos

Son roles ya definidos en las bases de datos. Traen permisos ya definidos y no pueden ser cambiados. Los roles específicos de SQL Server 2008 en adelante se explican a continuación.

db_owner
pueden realizar todas las actividades de configuración y mantenimiento en la base de datos y también pueden quitar la base de datos en SQL Server.

<<
>>

Roles Fijos

db_securityadmin
pueden modificar la pertenencia a roles y administrar permisos.

db_accessadmin
pueden agregar o quitar el acceso a la base de datos

db_backupoperator
pueden crear copias de seguridad de la base de datos.

db_ddladmin
pueden ejecutar cualquier comando del lenguaje de definición de datos (DDL) en una base de datos.

<<
>>

Roles Fijos

db_datawriter
pueden agregar, eliminar o cambiar datos en todas las tablas de usuario.

db_datareader
pueden leer todos los datos de todas las tablas de usuario.

db_denydatawriter
no pueden agregar, modificar ni eliminar datos de tablas de usuario de una base de datos.

db_denydatareader
no pueden leer datos de las tablas de usuario dentro de una base de datos.

<<
>>

roles definidos por el usuario


Son roles que el usuario puede crear y asignarle permisos específicos.

Crear roles
CREATE ROLE nombreRol [AUTHORIZATION nombreDueño];
nombreDueño es el usuario o rol de la base de datos a quien pertenece el nuevo rol. Si no se especifica el usuario, se tomará como dueño a quien crea el rol
CREATE ROLE buyers AUTHORIZATION BenMiller;

<<
>>

agregar y eliminar usuarios a rol

Agregar usuarios

ALTER ROLE Sales ADD MEMBER Barry;

Eliminar usuarios
ALTER ROLE nombreRol DROP MEMBER nombreUsuario;

ALTER ROLE Sales DROP MEMBER Barry;

Permisos

Los permisos sirven para brindar seguridad a la base de datos. Para ello restringe el acceso a ciertas funcionalidades a ciertos grupos de usuarios (roles).
Los conjuntos de permisos asignados a roles los heredan todos los miembros del rol. Es más fácil agregar o quitar usuarios de un rol que volver a crear conjuntos de permisos distintos para cada usuario.

<<
>>

Tipos de Permisos

ALTER
Permite cambiar las propiedades de un elemento determinado. También permite modificar, crear o quitar cualquier elemento protegible que esté contenido en ese ámbito. Por ejemplo, el permiso ALTER en un esquema incluye la posibilidad de crear, modificar y quitar objetos del esquema.

CONTROL
Dispone de todos los permisos definidos para el elemento protegible.

DELETE
Permite eliminar datos de una tabla.

EXECUTE
Permite ejecutar scripts externos, procedimientos, funciones escalares y de agregado.

<<
>>

Tipos de Permisos

INSERT
Permite cambiar las propiedades de un elemento determinado. También permite modificar, crear o quitar cualquier elemento protegible que esté contenido en ese ámbito. Por ejemplo, el permiso ALTER en un esquema incluye la posibilidad de crear, modificar y quitar objetos del esquema.

RECEIVE
Dispone de todos los permisos definidos para el elemento protegible.

REFERENCES
Utilizado para colas de Service Broker

SELECT
Permite seleccionar registros de una tabla.

<<
>>

Tipos de Permisos

TAKE OWNERSHIP
Permite al receptor del permiso tomar propiedad del elemento protegible para el que se concede este permiso.

UPDATE
Permite actualizar datos de una tabla.

VIEW CHANGE TRACKING
Permite visualizar los cambios que se han realizado en las tablas a través del change tracking.

VIEW DEFINITION
Permite al receptor obtener acceso a los metadatos.

<<
>>

Instrucciones de Permiso

Grant
Concede un permiso.

GRANT tipoPermiso ON elementoAObtenerPermiso TO usuario/rol;

GRANT SELECT ON employees TO Barry;

Revoke
Revoca un permiso.

REVOKE tipoPermiso ON elementoAObtenerPermiso FROM usuario/rol;

REVOKE INSERT ON employees FROM Barry;

Deny
DENY revoca un permiso de manera que no pueda ser heredado.
REVOKE tipoPermiso ON elementoAObtenerPermiso TO usuario/rol;
REVOKE UPDATE ON employees TO Barry;

La mayoría de los datos almacenados en las bases de datos, son obtenidos a través de aplicaciones. De igual manera los datos almacenados deben ser mostrados a los usuarios para su uso: reportes, gráficos, análisis, entre otros.
Por ello es importante poder conectar nuestra aplicación a una base de datos. Para el ejemplo se realizará la conexión de una aplicación realizada en C# a una base de datos SQL Server mediante el Framework .NET.
