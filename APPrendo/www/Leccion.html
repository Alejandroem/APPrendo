Introduccion

Que es SQL?

  • SQL significa Lenguaje Estructurado de Consultas por sus siglas en ingles
  • SQL permite acceder y manipular bases de datos
  • SQL es un estandar ANSI

Que puede hacer SQL?

  • Puede ejecutar consultas a una base de datos
  • Puede devolver informacion de una base de datos
  • Puede insertar registros en una base de datos
  • Puede actualizar registros en una base de datos
  • Puede eliminar registros en una base de datos
  • Puede crear nuevas bases de datos
  • Puede crear nuevas tablas en una base de datos
  • Puede crear procedimientos almacenados en una base de datos
  • Puede crear vistas en una base de datos
  • Puede colocar permisos en tablas, procedimientos y vistas

Que es RDBMS?

  • RDBMS significa Sistema de Gestion de Bases de Datos Relacionales por sus siglas en ingles
  • La informacion en un RDBMS es almacenada en objetos llamados tablas
  • Una tabla es una coleccion de datos relacionados y esta formada por filas y columnas

Llave Primaria

  • Asegura que los registros sean unicos evitando ambiguedad
  • Puede ser una o varias columnas
  • Se recomienda sea un numero autoincrementable
  • Se utiliza para hacer referencia a otras tablas
  • No puede ser nula
  • Se sugiere que no se cambie
  • Si se cambia las tablas referenciadas deben ser actualizadas

paso 3: Crear relaciones entre tablas

Relación: Es una asociación entre dos tabla. Muestra la interacción que hay entre ellas.


Relación uno a muchos


Un registro en una tabla A se relaciona con cero o muchos registros en una tabla B. Pero los registros de B solamente se relacionan con un registro en A.


Ej. Un profesor puede impartir varias clases pero una clase puede ser impartida por un unico profesor

Relacion muchos a muchos


Una tabla A se puede relacionar con cero o con muchos registros en B y viceversa. Este tipo de relación no puede existir por lo que debe ser disuelta.


Para ello se crea una tabla intermedia con las llaves primarias de las tablas a disolver. Se pueden agregar otros atributos a la nueva tabla.


Ej. Una orden puede incluir varios productos y un producto puede estar incluido en varias ordenes, por ello se crea una tabla, la cual llamaremos detalles de la orden.

Relacion uno a uno


Un registro de una tabla A se relaciona con solo un registro en una tabla B y viceversa.


Ej. Un producto puede tener un detalle y ese detalle pertenece unicamente a un producto

Son consultas que se realizan para crear una base de datos y manipular su estructura, es decir crear, modificar y eliminar sus tablas.

>>

Sintaxis

Ejemplo

>>

Constraints

Especifican reglas para los valores en una tabla.


Not Null


Una columna no puede tener valores nulos

Unique


Asegura que cada registro en esa columna sea único. Puede existir más de una.

Primary Key


Identifica un registro de manera única. Esto ayuda a encontrar el registro de manera más fácil y rápida. No puede repetirse ni ser nula.



Una llave primaria puede estar compuesta por varias columnas


Foreign Key


Hace referencia de una tabla a otra. Una llave foránea en una tabla apunta a la llave primaria de otra tabla.

DML (Data Manipulation Language)

¿Qué es DML?

Son consultas que se realizan para manipular los datos dentro de la base de datos, es decir insertar datos a la base de datos, eliminarlos, actualizarlos o seleccionarlos.

>>

Insertar Datos

Sintaxis
INSERT INTO nombreTabla (columna1, columna2, columna3,...)
VALUES(valor1, valor2, valor3,...);

Ejemplo
INSERT INTO Customers (
CustomerName, ContactName, Address, City, PostalCode, Country)
VALUES
('Cardinal','Tom B. Erichsen','Skagen 21','Stavanger','4006','Norway');


Para insertar datos tipo varchar, estos van encerrados en comillas simples. Los valores se irán ingresando en el mismo orden en el que se definieron las columnas; en el ejemplo ‘Cardinal’ se almacenará en CustomerName, y ‘4006’ en PostalCode.

<<
>>

Eliminar datos

Todos los datos

Sintaxis
DELETE FROM nombreTabla;

Ejemplo
DELETE FROM customers;

Datos específicos

Sintaxis
DELETE FROM nombreTabla WHERE
nombreColumna = algunValor;

Ejemplo
DELETE FROM customers
WHERE CustomerName= 'Alfreds Futterkiste' AND ContactName='Maria Anders'; ;

En este ejemplo se eliminarán todos los clientes cuyo nombre del cliente sea ‘Alfreds Futterkiste’ y el nombre del contacto sea ‘María Anders’

<<
>>

Actualizar Datos

Sintaxis
UPDATE nombreTabla
SET columnaModificar1 = valor1, columnaModificar2 = valor2,...
WHERE algunaColumna = valor;

Ejemplo
UPDATE Customers
SET ContactName = 'Alfred Schmidt',
City = 'Frankfurt'
WHERE CustomerID = 1;

Si no se coloca la condición WHERE se actualizarán todos los registros en la tabla.
En el ejemplo se asigna el valor ‘Alfred Schmidt’ a los registros de la columna ContactName y el valor ‘Frankfurt’ a los registros en la columna city de la tabla Customers cuando el CustomerID sea igual a 1.
